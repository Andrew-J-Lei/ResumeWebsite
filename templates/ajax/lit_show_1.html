Overview
A friend, Christopher Gilmore, and I wrote a program that makes addressable LEDs music reactive. What started off as a small Christmas gift for my sister became increasingly advanced as Chris and I added more features, changed code structures, and improved core mechanics.

Relative to some of my other projects, which had more defined goals for success, the success of this project was almost entirely subjective and therefore more abstract. As my partner and I both simply wanted it to look good, it forced us to be more creative and to test out more possibilities rather than to research specific solutions to problems.

This was also my most closely collaborative project. That is to say, I've rarely had a partner that would be explicitly working on the same parts of code as I. This forced both of us to more closely understand each others' code and, when adding new features, to make sure that there isn't a loss of functionality and that a clean merge can be made between different branches of code.
